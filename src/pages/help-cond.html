Conditional Expressions

Conditional expressions allow you to perform logical tests on information in the program such as is 5 less than 6 or is 4 equal to twice the value of some variable. Conditional expressions produce values in the program called Boolean values. Boolean values are truth values (i.e. yes/no, true/false). The Simulator defines two keywords for literal Boolean values that you can use in expressions. They are true and false. They aren't really useful by themselves, but they represent the two possibile Boolean values used in conditional expressions (that is to say that every conditional expression evaluates down to either true or false).

Conditional expressions are used in control structures such as if and while blocks. You cannot use them in operation blocks. This means a variable cannot store a Boolean value.

Just like operation expressions that use numbers, conditional expressions have operators that use either Boolean values or numbers to produce a Boolean value. Here they are, in order of precedence:

5 not A evaluates the logical negation of A; i.e. if A is false it becomes true and vice versa A must be a Boolean value
4 A < B evaluates to true if A is less than B; false otherwise A and B must be numbers
4 A <= B evaluates to true if A is less than or equal to B; false otherwise A and B must be numbers
4 A > B evaluates to true if A is greater than B; false otherwise A and B must be numbers
4 A >= B evaluates to true if A is greater than or equal to B; false otherwise A and B must be numbers
3 A == B evaluates to true if A is equal to B; false otherwise A and B must be numbers
3 A <> B evaluates to true if A is not equal to B; false otherwise A and B must be numbers
2 A and B evaluates to true if A is true and B is true; false otherwise A and B must be Boolean values
1 A or B evaluates to true if one or more of A or B is true; false otherwise A and B must be Boolean values

It is important to note that the logical or operator evaluates to true if any of its operands is true. This is different from the common English understanding of the word or, which means exclusively one or the other (this is called exclusive-or or xor).
