Input Blocks

A computer program wouldn't be very useful if it couldn't read user input. Input blocks allow you to prompt the user for an input value and assign those values to variables. Input blocks use format strings to specify the input operation (just like output blocks). When a variable is specified in a format string, the input block requests input from the terminal emulator at the bottom of the screen. The user then enters input and the block assigns it to the specified variable(s).

Non-variable text of the input block's format string will be printed up until a variable is specified. This is true for each variable in the format string. For example, the following format string will print Enter number:  and then read input that will be assigned to variable a.

Enter number: %a

This produces the following (given input 5):

Enter number: 5

Here's a more complicated format string example:

Enter %a and then %b.

This produces the following (given input 5 and 8):

Enter 5
and then 8
.

It is generally not a good idea to have part of the output prompt after a variable in an input block format string. As you can see, the user entering a line makes the message span multiple lines.

Finally, there are a couple of things to know about how input is read and processed by the block:

  • Every line of input is split up into whitespace-separated tokens. The block then converts each token into a number and assigns it to a variable. Since you can specify more than one variable in a format string, the block can assign multiple values from one line of input. If the user didn't supply enough input on a line, another line is requested (and so on until all of the required input has been read).
  • Extra tokens read by the block are not available to future blocks. In other words, each line of input is available only to the block that requested it.
  • The Simulator only operates on numbers; therefore the user should not enter any non-numeric input string. If they do, the Simulator will fail and stop execution.
  • If an empty line is read, the block will do nothing and request another line.
